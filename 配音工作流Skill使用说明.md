# 配音工作流 Skill 使用说明

## 安装

### 方式一：GitHub 链接安装（推荐）

把下面这句话发给 Agent：

```
安装 https://github.com/wayne3991/peiyin_workflow/raw/main/peiyin-workflow.zip
```

### 方式二：zip 拖入安装

把 `peiyin-workflow.zip` 拖入对话框，说：

| 工具 | 说的话 |
|------|--------|
| **CodeBuddy** | 「安装为 CodeBuddy Skill」 |
| **其他 AI 工具** | 「解压并使用里面的配音工作流」 |

> 其他工具包括：Cursor、Claude Code、Codex、Windsurf 等。

---

## 开始制作一条视频

### 1. 把文案 MD 放入输入文件夹

将你要配音的 MD 文档放入 `克隆配音文案/` 文件夹。

### 2. 启动配音流程

在对话框输入对应启动词：

| 输入 | 说明 |
|------|------|
| `配音` | 每一步做完会暂停让你确认 |
| `一键配音` | 全程自动，选好文件后直接出成品 |

### 3. 选择文件

- 一个文件自动选中，多个文件列表选择

### 4. 等待完成

全部完成后，在 `output/原文件名_日期/` 里拿到成品。

---

## 启动方式（四种）

| 输入 | 模式 | 说明 |
|------|------|------|
| `配音` | 交互配音 | 每步暂停确认 |
| `一键配音` | 自动配音 | 全自动执行 |
| `测试模式` | 交互测试 | 生成测试文档，每步确认 |
| `一键测试` | 自动测试 | 全自动，最后问是否删除 |

---

## 轻量模式（16GB 内存 Mac 专用）

防止 TTS 闪退的常驻开关。状态保存在工作目录下，重启后保持。

| 输入 | 说明 |
|------|------|
| `开启轻量模式` | 打开优化，内存占用降低约 40% |
| `关闭轻量模式` | 恢复正常参数 |

**影响**：长段回答会多几个轻微换气停顿，短文本无影响，音质不变。

**每次配音前 Skill 会自动提示当前状态**：`🔔 当前轻量模式：开启中 / 关闭中`

|------|----------|----------|
| 角色 | 巴菲特、芒格、胡博 | 胡博 + 一位大师 |
| 流程 | 6 步 | 5 步 |

## 音频生成规则

音频按**角色说话次数**切段，不是按自然段。

比如一篇股东大会文稿里的对话流程：

```
巴菲特：第一句话（钩子）
胡博：引出问题
巴菲特：长篇回答
芒格：简短补充
胡博：小结
```

→ 最终生成 **5 段音频**：
`01_巴菲特.wav`、`02_胡博.wav`、`03_巴菲特.wav`、`04_芒格.wav`、`05_胡博.wav`

> 巴菲特长篇回答是一整段音频，不会按自然段拆碎。准备初始文稿时，确保每个角色的每段台词是一个完整自然的发言单元。

## 成品里有什么

| 产物 | 说明 |
|------|------|
| `_股东大会文字稿优化.md` | 对话体文案 |
| `_字幕处理.md` | 分句字幕（可直接导入剪辑软件） |
| `_文字总结.md` | 视频开头钩子 + 结尾总结 + 短视频标题 |
| `AI配音/01_角色名.wav` 等 | 克隆好的音频文件 |

---

## 关键规则

- 声音文件已内置在 zip 包中
- 英文名去中点（如「查理·芒格」→「查理芒格」）
- 黄金 3 秒：开头 3 秒抛出核心观点或悬念
- 标题 ≤15 字，约 10 个
- 交互模式步进确认，一键模式全程自动
